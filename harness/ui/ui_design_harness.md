# ui_design_harness.md

## 1. Visual System
- 색상 토큰, 타이포, 간격, 그림자, 반경을 분리해 정의한다.
- 임시 색상 하드코딩을 금지한다.

## 2. Layout System
- 정보 우선순위를 화면 구조에 반영한다.
- 헤더 / 본문 / 보조 패널 / CTA의 역할을 명확히 나눈다.

## 3. Component System
- 버튼, 입력창, 카드, 모달, 테이블, 탭, 배지 상태를 정의한다.
- 기본/호버/포커스/활성/비활성/오류 상태를 문서화한다.

## 4. Interaction System
- 로딩, 빈 상태, 오류 상태, 성공 피드백을 설계한다.
- 모션은 정보 전달을 돕는 범위로 제한한다.

## 5. Accessibility
- 대비, 포커스, 키보드 접근성, 라벨링을 점검한다.

## 6. Responsive
- 브레이크포인트마다 정보 밀도와 우선순위를 조정한다.

## 7. UX Evaluation
- 완료 시간, 실수 빈도, 재시도 지점, 이탈 지점을 측정한다.
